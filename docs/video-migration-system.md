# Техническая документация: Система миграции видео в формат HLS

## Обзор системы

Система миграции видео предназначена для конвертации существующих видеофайлов в формат HTTP Live Streaming (HLS), который обеспечивает адаптивное потоковое воспроизведение видео. Система включает в себя серверную часть для конвертации видео и пользовательский интерфейс для управления процессом миграции.

## Архитектура системы

### Компоненты системы

1. **Конвертер видео** (`src/lib/video/converter.ts`)
   - Отвечает за скачивание, конвертацию и загрузку видео
   - Использует FFmpeg для конвертации в формат HLS
   - Создает превью (thumbnail) для видео
   - Загружает файлы в Yandex Object Storage

2. **API для управления миграцией** (`src/app/api/management/videos/migration/route.ts`)
   - Предоставляет эндпоинты для получения списка видео и запуска миграции
   - Обрабатывает запросы на миграцию выбранных видео

3. **Интерфейс управления миграцией** (`src/app/[locale]/management/media/migration/page.tsx`)
   - Отображает список видео с их статусом миграции
   - Позволяет выбирать видео для миграции
   - Отображает прогресс миграции и ошибки
   - Использует компоненты HeroUI для пользовательского интерфейса

4. **База данных**
   - Хранит информацию о видео в таблице `ProjectMedia`
   - Включает поля для хранения URL, статуса миграции и метаданных

# Техническая документация: Система миграции видео в формат HLS

## Обзор системы

Система миграции видео предназначена для конвертации существующих видеофайлов в формат HTTP Live Streaming (HLS), который обеспечивает адаптивное потоковое воспроизведение видео. Система включает в себя серверную часть для конвертации видео и пользовательский интерфейс для управления процессом миграции.

## Архитектура системы

### Компоненты системы

1. **Конвертер видео** (`src/lib/video/converter.ts`)
   - Отвечает за скачивание, конвертацию и загрузку видео
   - Использует FFmpeg для конвертации в формат HLS
   - Создает превью (thumbnail) для видео
   - Загружает файлы в Yandex Object Storage

2. **API для управления миграцией** (`src/app/api/management/videos/migration/route.ts`)
   - Предоставляет эндпоинты для получения списка видео и запуска миграции
   - Обрабатывает запросы на миграцию выбранных видео

3. **Интерфейс управления миграцией** (`src/app/[locale]/management/media/migration/page.tsx`)
   - Отображает список видео с их статусом миграции
   - Позволяет выбирать видео для миграции
   - Отображает прогресс миграции и ошибки
   - Использует компоненты HeroUI для пользовательского интерфейса

4. **База данных**
   - Хранит информацию о видео в таблице `ProjectMedia`
   - Включает поля для хранения URL, статуса миграции и метаданных

### Схема работы

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Интерфейс  │     │     API     │     │  Конвертер  │
│  миграции   │────▶│  миграции   │────▶│    видео    │
└─────────────┘     └─────────────┘     └─────────────┘
       │                   │                   │
       │                   │                   │
       │                   │                   ▼
       │                   │           ┌─────────────┐
       │                   │           │   FFmpeg    │
       │                   │           └─────────────┘
       │                   │                   │
       │                   ▼                   ▼
       │           ┌─────────────┐     ┌─────────────┐
       └──────────▶│    База     │◀────│   Yandex    │
                   │   данных    │     │   Storage   │
                   └─────────────┘     └─────────────┘
```

## Технические детали

### Конвертер видео

Конвертер видео выполняет следующие шаги:

1. **Скачивание видео**
   - Скачивает исходное видео по URL
   - Сохраняет во временную директорию

2. **Конвертация в HLS**
   - Использует FFmpeg для конвертации видео в формат HLS
   - Создает манифест-файл (master.m3u8) и сегменты (segment_XXX.ts)
   - Настройки кодирования: libx264, preset fast, CRF 22, AAC аудио

3. **Создание превью**
   - Создает превью (thumbnail) из кадра на второй секунде видео
   - Сохраняет как JPEG-изображение

4. **Загрузка в хранилище**
   - Загружает все файлы (манифест, сегменты, превью) в Yandex Object Storage
   - Устанавливает публичный доступ к файлам
   - Возвращает URL манифеста и превью

### API миграции

API миграции предоставляет два основных эндпоинта:

1. **GET /api/management/videos/migration**
   - Возвращает список всех видео с их статусом миграции
   - Фильтрует видео по типу "video"
   - Включает информацию о проекте, к которому относится видео

2. **POST /api/management/videos/migration**
   - Принимает массив ID видео для миграции
   - Запускает асинхронный процесс миграции для каждого видео
   - Возвращает подтверждение о начале миграции

### Интерфейс управления

Интерфейс управления миграцией предоставляет следующие возможности:

1. **Просмотр списка видео**
   - Отображает все видео с их названиями, URL и статусом
   - Показывает статистику по всем видео (общее количество, ожидающие, в процессе, завершенные, с ошибками)
   - Использует компоненты HeroUI (Card, Table, Chip, Progress и др.)

2. **Управление миграцией**
   - Позволяет выбирать видео для миграции с помощью чекбоксов
   - Запускает процесс миграции для выбранных видео
   - Отображает прогресс миграции и ошибки

3. **Просмотр видео**
   - Позволяет просматривать видео в модальном окне
   - Отображает информацию о видео и его статусе
   - Предоставляет возможность воспроизведения видео

### База данных

Таблица `ProjectMedia` включает следующие поля для поддержки миграции видео:

- `id`: Уникальный идентификатор медиафайла
- `projectId`: ID проекта, к которому относится видео
- `type`: Тип медиафайла (video, image, etc.)
- `url`: URL медиафайла
- `thumbnailUrl`: URL превью для видео
- `isCover`: Флаг, указывающий, является ли медиафайл обложкой
- `isMainVideo`: Флаг, указывающий, является ли видео основным
- `metadata`: JSON-поле для хранения дополнительной информации, включая:
  - `migrationInProgress`: Флаг, указывающий, что миграция в процессе
  - `migrationProgress`: Процент выполнения миграции (0-100)
  - `migrationError`: Сообщение об ошибке, если миграция не удалась

## Процесс миграции

### Шаги миграции

1. **Выбор видео для миграции**
   - Администратор выбирает видео для миграции через интерфейс
   - Нажимает кнопку "Мигрировать выбранные"

2. **Запуск миграции**
   - Система отправляет запрос на API с ID выбранных видео
   - API запускает асинхронный процесс миграции для каждого видео
   - Статус видео меняется на "processing"

3. **Конвертация видео**
   - Конвертер скачивает исходное видео
   - Конвертирует его в формат HLS с помощью FFmpeg
   - Создает превью для видео
   - Загружает все файлы в Yandex Object Storage

4. **Обновление базы данных**
   - После успешной конвертации система обновляет запись в базе данных
   - Обновляет URL видео на URL HLS-манифеста
   - Добавляет URL превью
   - Устанавливает статус "completed"

5. **Обработка ошибок**
   - Если возникает ошибка, система записывает ее в метаданные видео
   - Устанавливает статус "failed"
   - Отображает ошибку в интерфейсе

## Настройка и конфигурация

### Переменные окружения

Система использует следующие переменные окружения:

- `YANDEX_ACCESS_KEY_ID`: Ключ доступа к Yandex Object Storage
- `YANDEX_SECRET_ACCESS_KEY`: Секретный ключ доступа к Yandex Object Storage
- `YANDEX_BUCKET_NAME`: Имя бакета в Yandex Object Storage
- `YANDEX_STORAGE_ENDPOINT`: Эндпоинт Yandex Object Storage
- `YANDEX_STORAGE_URL`: URL для доступа к файлам в Yandex Object Storage
- `TEMP_DIR`: Директория для временных файлов (по умолчанию "/tmp")

### Зависимости

Для работы системы требуются следующие зависимости:

- **FFmpeg**: Для конвертации видео в формат HLS
- **Node.js**: Для запуска серверной части
- **AWS SDK**: Для работы с Yandex Object Storage (совместим с S3 API)
- **Next.js**: Для серверной и клиентской части приложения
- **React**: Для пользовательского интерфейса
- **HeroUI**: Для компонентов пользовательского интерфейса
- **Sonner**: Для отображения уведомлений

## Рекомендации по использованию

### Оптимальное использование

1. **Планирование миграции**
   - Мигрируйте видео в периоды низкой нагрузки на сервер
   - Мигрируйте небольшие партии видео за раз (5-10)
   - Следите за использованием дискового пространства на сервере

2. **Мониторинг**
   - Регулярно проверяйте статус миграции
   - Обращайте внимание на ошибки и повторяйте миграцию при необходимости
   - Следите за использованием хранилища в Yandex Object Storage

3. **Производительность**
   - Конвертация видео требует значительных ресурсов CPU
   - Убедитесь, что сервер имеет достаточно ресурсов для конвертации
   - Рассмотрите возможность использования выделенного сервера для миграции больших объемов видео

## Устранение неполадок

### Типичные проблемы и решения

1. **Ошибка доступа к Yandex Object Storage**
   - Проверьте правильность ключей доступа
   - Убедитесь, что бакет существует и доступен
   - Проверьте права доступа к бакету

2. **Ошибки FFmpeg**
   - Убедитесь, что FFmpeg установлен и доступен в системе
   - Проверьте права доступа к временной директории
   - Проверьте, что исходное видео доступно и имеет поддерживаемый формат

3. **Проблемы с временными файлами**
   - Убедитесь, что временная директория существует и доступна для записи
   - Проверьте наличие достаточного дискового пространства
   - Регулярно очищайте временную директорию от старых файлов

4. **Проблемы с UI-компонентами**
   - Убедитесь, что используются правильные импорты из библиотеки HeroUI
   - Проверьте совместимость версий HeroUI и Next.js
   - При обновлении библиотек UI следите за изменениями в API компонентов

Система разработана с учетом масштабируемости и производительности, позволяя эффективно управлять процессом миграции через удобный пользовательский интерфейс.
